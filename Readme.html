Release notes

Lots of extra features, lots of bug fixes and, almost certainly, lots of juicy new bugs. Notably: web based admin now pretty much works, GD 2 and ImageMagick may be used for better thumbnail generation.

Installation

singapore should work 'out of the box' but there are many things that can be done to customize it to your needs. Here are some:

Things you MUST do:

  • Extract all the files in the archive, conserving the directory hierarchy, to a directory accessible by your web server.

Things you SHOULD do:

  • Add some galleries and images into the galleries/ directory (and remove the example ones provided). More info.
  • Set file permissions. This part is dependent on your operating system and web server setup and can seem quite daunting to a beginner. Your best bet, if you need help, is to contact your server administrator. Here is the list of files and directories that need to be writable by the web server:
    • cache/
    • galleries/ and all subdirectories and files
    • includes/adminusers.csv
    • logs/ unless logging is disabled
  • Log into the admin section and change the password. The username is admin and the default password is password.
    NOTE: a future release will allow multiple admin users with different passwords.
  • If your PHP installation uses GD version 2 or later or you have access to ImageMagick then you can get much nicer generated thumbnails by changing the option in includes/class_configuration.php.

Things you MIGHT do:

  • Edit includes/header.php and includes/footer.php or use your own custom header and/or footer files by editing the paths in includes/class_configuration.php.
  • Change the secret string value in includes/class_configuration.php. This is not essential but it makes it harder for someone to fake an admin session thereby gaining unauthorised access to your galleries.
  • Edit the values in includes/class_configuration.php. The file is commented and should be self-explanatory. For most purposes the default values will be fine.
  • Make custom "themes" using CSS. More info.

Managing your galleries

If using the info in file name system then all management can be done directly on the files using, for example, an ftp client. To create new galleries simply create a new subdirectory of galleries/. To add, move or delete images in a gallery simply add, move or delete the image files in the appropriate directory.

If you use the admin mode to edit your galleries or images it will automatically create the relevant metadata files. If these files are deleted singapore will revert to using info in file name.

There is currently no support for bulk image upload via the admin interface. You are recommended to use FTP to upload your images and then use the admin mode to add image information and for occasional additions and deletions.

Info in file name

If singapore doesn't find a metadata file in a gallery it will try to extract some sensible information from the image file and directory names as follows:

Underscores (_) are converted to spaces and the file extension is discarded by removing all characters after the last . (dot) in the filename. If a filename or directory name contains the sequence '_-_' (underscore hyphen underscore) or ' - ' (space hyphen space) then the name will be split on this separator. The first half will be interpreted as the artist name and the second half will be interpreted as the gallery or image name depending on context. If a separator is not found then the file or directory name is interpreted as the image or gallery name, respectively.

For example Holiday_Snaps/Tamlyn_Rhodes_-_Pretty_picture_of_Sunset.jpg will be translated into an image called "Pretty picture of Sunset" by "Tamlyn Rhodes" in a gallery called "Holiday Snaps".
